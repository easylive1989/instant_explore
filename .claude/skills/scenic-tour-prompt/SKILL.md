# scenic-tour-prompt

把「人物圖 + 同一景點的多張圖 + 使用者提供的景點故事」轉成一份**全英文、結構化的 Seedance 2.0 影片提示詞**。影片內容＝人物在景點中遊覽、用肢體與鏡頭呈現導覽介紹感，**純視覺敘事、無聲、畫面無文字**。

提示詞撰寫規律與黃金公式見 `references/prompt_template.md`，**組裝前務必讀**。

## 核心原則

- **看圖萃取，不要憑空想像**：人物外貌與每個景點場景的細節，一律從使用者提供的圖檔讀出來，再寫進提示詞。
- **純視覺敘事**：人物的「介紹」只能靠走動、駐足、手勢指向、回望、鏡頭帶到景物來表達。**不得**寫旁白、對白、字幕、畫面文字。
- **一致性優先**：人物極易在 AI 影片中變形，每次都要放一致性約束咒語並用 `@image` 指代首幀人物參考。
- **一張景點圖 ≈ 一個分鏡段**：用分秒時間軸把多張景點圖串成一段連續遊覽。

## 流程

### 1. 收集輸入（缺項主動問，一次問一項）

必要：
- **人物圖**：1 張圖檔路徑。
- **景點圖**：同一景點的多張圖檔路徑。
- **景點名稱**。
- **要介紹的故事/重點**：使用者用文字提供（此 skill 不自行編造景點歷史）。

可選（有預設）：
- **畫面比例**：預設 `vertical_9x16`（直式，社群短影音）。
- **影片總長**：不指定時，依景點圖數量估，每場景約 3–5 秒。
- **風格**：預設 `cinematic, realistic`，可改 `anime` 等。

### 2. 看圖萃取

- **讀人物圖** → 寫出一致性咒語：臉部、髮型、服飾、配件、體型比例、色彩（見 template 的咒語庫）。
- **逐張讀景點圖** → 為每張記下：場景主體、光線/氛圍、空間結構，以及**適合的運鏡**（例：開闊全景→環繞或廣角；長廊→跟拍推進；細節→淺景深特寫）。

### 3. 規劃分鏡（timeline）

- 依景點圖張數與總長，切成分秒段落（`[0-4s] / [4-8s] ...`）。
- 每段＝一張景點圖場景 + 人物動作 + 鏡頭運動，並把使用者的故事重點對應到合適的段落（用視覺呈現該重點，而非文字說明）。
- 段落間維持人物移動的連續性，營造「一路遊覽」的感覺。

### 4. 套黃金公式組裝（讀 `references/prompt_template.md` 後）

順序：YAML 參數區塊 → 整體風格氛圍 → 一致性約束（含 `@image1` 人物參考）→ 分秒時間軸分鏡 → 相機軌跡與技術參數收尾。

### 5. 輸出

- 全英文 Seedance 2.0 提示詞，以 code block 呈現方便複製。
- 結尾用一兩句中文摘要這支影片的分鏡走向，方便你快速確認方向是否正確（這段只在對話裡，不寫進提示詞）。

## 常見錯誤

| 錯誤 | 修正 |
|---|---|
| 沒看圖就憑景點名稱亂寫場景 | 一定要實際讀每張圖檔再描述 |
| 寫進旁白／字幕／畫面文字 | 純視覺敘事，移除所有文字與聲音元素 |
| 漏掉一致性約束 → 人物變形 | 每次都加咒語 + `@image1` 參考 |
| 把多張圖塞進同一段 | 一張景點圖對應一個分鏡段 |
| 提示詞用中文輸出 | 提示詞一律全英文（模型最穩） |
| 故事用文字卡呈現 | 改用人物手勢、鏡頭帶景物等視覺方式表達 |
